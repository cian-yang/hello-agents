# 低代码平台价值

1、降低技术门槛

2、提升开发效率

3、提供更优的可视化与可观测性

4、标准化与最佳实践沉淀：优秀的低代码平台通常会内置许多行业内的最佳实践。

意义：从繁琐的底层实现中解放出来，更**专注于智能体“思考”与“行动”的逻辑本身，从而更快、更好地将创意变为现实**。

# 常见低代码平台

三个各具代表性的平台：Coze、Dify和 n8n

- Coze：主打零代码/低代码的 Agent 的构建体验，让不具备编程背景的用户也能轻松创造。极其友好的可视化界面
- Dify：开源的、功能全面的 LLM 应用开发与运营平台，为开发者提供从原型构建到生产部署的一站式解决方案。融合了后端服务和模型运营的理念
- n8n：开源工作流自动化工具。n8n 的强项在于“连接”。

# Dify

Dify 是一个开源的大语言模型（LLM）应用开发平台，融合了后端即服务（BaaS） 和 LLMOps 理念，为从原型设计到生产部署提供全流程支持

采用分层模块化架构，分为数据层、开发层、编排层和基础层，各层解耦便于扩展

核心优势

- 全栈式开发体验
- 低代码与高扩展性的平衡
- 企业级安全与合规
- 丰富的工具集成能力
- 活跃的开源社区

局限

- 学习曲线较陡
- 性能瓶颈
- 多模态支持不足：当前主要以文本处理为主
- 企业版成本较高
- API 兼容性问题：Dify 的 API 格式不兼容 OpenAI，可能限制与某些第三方系统的集成

# n8n

n8n 的核心身份是一个通用的工作流自动化平台，而非一个纯粹的 LLM 应用构建工具

使用 n8n 构建智能应用时，我们实际上是在设计一个更宏大的自动化流程，而大语言模型只是这个流程中的一个（或多个）强大的“处理节点”。

n8n 的世界由两个最基本的概念构成：**节点 (Node)和工作流 (Workflow)**

- **节点 (Node)**：工作流中执行具体操作的**最小单元**。你可以把它想象成一个具有特定功能的“积木块”。节点大致可以分为两类：
  - **触发节点 (Trigger Node)**：整个工作流的起点，负责启动流程。
  - **常规节点 (Regular Node)**：负责处理具体的数据和逻辑。
- **工作流 (Workflow)**：工作流是由**多个节点连接而成**的自动化流程图。
  - 定义了数据从触发节点开始，如何一步步地在不同节点之间传递、被处理，并最终完成预设任务的完整路径。
  - 数据在节点之间以结构化的 JSON 格式进行传递，这使得我们可以精确地控制每一个环节的输入和输出。

**n8n 的真正威力在于其强大的“连接”能力**。

优势：

- 开发效率
- 功能强大且高度集成：n8n 拥有丰富的内置节点库，可以轻松连接像 Gmail、Google Gemini 等数百种常见服务。
- 部署运维：n8n 支持 **私有化部署**，并且也是目前相对比较简单且能部署完整版项目的私有化Agent方案

局限性：

- 调试与错误处理的相对繁琐：当工作流变得复杂时
- **内置存储的非持久性（最大局限）**：n8n 服务一旦重启，所有对话历史和知识库都将丢失。在实际部署时，必须将其替换为如 Redis、Pinecone 等外部持久化数据库，这也会增加了额外的配置和维护成本。
- 版本控制和多人协作不如传统代码成熟
- 性能
